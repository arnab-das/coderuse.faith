Deploying Faith

Configured CI with Bitbucket pipeline and deployed the site to GitHub pages.

Aug 30, 2017 • Arnab Das • #100-words

Thought that, GitHub pages will be able to handle the Jekyll page generation and deploy will be done automatically. Alas. My assumption came out to be incorrect. GitHub pages’ Jekyll generator works half heartedly only for the index pages. Not for others. So, deleted the old repo and configured CI as in coderuse.com in Bitbucket. Free CI on a need basis. Gives 500 mins for free to build on post check-in hook. Created password token and added in the pipeline variables. This is going to be the first check-in after the set-up. Let’s see if it works. Or will have to do more check-ins changing very little in files. Added the random Git commit messages. So, relieved that don’t have to come with new messages every time. Just git random.

Update I may be wrong about GitHub pages. In my case the constructed url’s were wrong. But I’ve to check again by creating a site again. For now it’s working perfectly with the Bitbucket pipelines.
