In software companies, the term brainstorming is quite popular, especially to the freshers, aka fresh out of engineering college. But before starting this topis, I should warn you that I posses a very negative view towards the activities, that are generally done for this act of brainstorming. Yes, this is an acting, atleast most of the time. Okay let’s try to put up a description of brainstorming. First rule of brainstorming, there has to be more than two people in that session, actually more the merrier. Second, there has to be one separate room, in which this act to be carried on. Third, the topic should be as much abstract as possible, so that the actors should be able to kill some time to decide the exact outcome of it. I’ve never seen any of this kind of sessions has ended in less than one hour. It goes on and goes on. And long time, some senior like project lead or manager chucks in to the room and just puzzle all the hitherto understood concepts. And then again the session starts from scratch again. In the services sector of software industry this is another sacred term. We are like the carpenters. Only want to do, feel happy to do a project from scratch. Whatever solution or opensource project there be, we always pitch for start from scratch. Bigger the organisation, more the rush for it. There is a business part of it also, but it appeals most to the developers. They don’t need to learn anything, don’t need to see any existing solution. Just brainstorm over the requirement and do something, release and then make the customer to opt for the maintainance period. Customer has to take the maintainance to get rid of the bugs in the from-scratch application. This is one sort of brainstorming needed to be done for client requirement. Another is for the free people, aka people who are in bench. The poor fellows have to perform, have to do some extra miracle, so they have to chime in any bogus ideas of managers.
