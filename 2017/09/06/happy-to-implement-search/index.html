Sep 06, 2017 • Arnab Das • #100-words

For the last couple of days, trying to implement a search functionality in this website. Now it’s complete. Feeling happy with this success. Implentation of this includes quite a few loose components. A search engine, a NodeJs app as an ACL, Nginx as reverse proxy and the Nginx configured with LetsEncrypt SSL.

First I tried to look for a simpler and easier solution. Got a few good posts also. The search can be done in the client side or in the server side or by configuring with Google Search Engine itself.

Configuring with Google Search engine is the simplest one. But the search result page redirects to Google’s domain. That’s not a very good experience. But for a quick and dirty fix, it’s quite a promising one.

Client side search is a very intuitive one. There are specific solutions for popular static site generators like Hugo, Hexo, Jekyll. This solution is acceptable for sites with preferably not-so-much content. Because it requires to generate a json file with all the posts available in the site. And search is done in the client side by JavaScript. With a large number of posts, even if downloading of the json file with gzip compression enabled don’t appeared to be a big problem but searching in the client side may result the browser unresponsive for a small duration. I don’t know, whether my site is going to be that big or not, but I prefer to on safe side and implementing the search in the server side, would require at-least some effort, and learning new things. I always like to learn new things.

For server side searching first I attracted to ParaIO. That seemed to be full of features also. But while configuring the search, realized I need some sort of ACL. Means I need a server. So, if I require one server, why not use Elastic Search on that server also. By that I will be able to use the same infrastructure in other sites also. Most of my sites are static.

So, took onen VPS. Deployed ElasticSearch. ElasticSearch is very light on run and easy to configure. Written one NodeJs app to parse and upload the posts to the ElasticSearch and one for the ACL. Installed Nginx and configured SSL with LetsEncrypt and reverse proxy to the ACL. Now the search is available for faith. Probably I will write one or multiple posts in my technical blog. But that is the task for future. For today, let’s call it a day.
